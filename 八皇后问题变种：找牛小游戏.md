# 问题描述

> [!question] Title
> 在 n×n 棋盘上，有 n 种颜色，每种颜色占据一个连通区域。
> 要在每个颜色区域内放置恰好一头"牛"，满足：
>
> 1. 不同牛不能在同一行
> 2. 不同牛不能在同一列
> 3. 不同牛不能四对角相邻

# 问题分析

## 核心数据分析

```cpp
using Pos = pair<int, int>;           // (行, 列)
using Candidates = vector<Pos>;       // 一个颜色的候选位置列表
using Available = vector<Candidates>; // 所有颜色的候选列表 (available
```

## 求解器 (Solver) 的类定义

```cpp
class Solver
{
public:
    Solver(int n, const Available &positionsByColor):
    n_(n), board_(n, vector<int>(n, 0)),
    initial_available_(positionsByColor) {}
    /*C++ 规则：
    成员在进入 {} 之前已完成初始化。
    不写在初始化列表里 → 先调默认构造（空 vector），
    再调operator=（拷贝所有元素），
    两步操作，中间的空 vector 纯属浪费。*/
   
    bool solve() { return dfs(0, initial_available_, 0); }
    // 求解入口，封装求解细节，以免参数输入错误
    const vector<vector<int>> &board() const { return board_; }
    // 不采用值传递，用两个const形成类内外只读的引用
    int n() const { return n_; }
    // 数据量小，采用值传递

private:
    static bool conflictsWith(int x, int y, int px, int py) {...}
    bool dfs(int depth, Available available, uint32_t placed_mask) {...}

    int n_;
    vector<vector<int>> board_;
    Available initial_available_;
};
```

## 复杂函数实现

### 1. static bool conflictsWith(int x, int y, int px, int py)

> [!quote] Parameter
>
> - ( x , y )：待验证的位置
> - ( px , py )：已放置的牛的位置

```cpp
    static bool conflictsWith(int x, int y, int px, int py)
    {
        if (x == px || y == py)
            return true; // 同行或同列
        int dx = abs(x - px);
        int dy = abs(y - py);
        return dx == 1 && dy == 1; // 四对角相邻
    }
```

### 2. bool dfs(int depth, Available available, uint32_t placed_mask)

> [!quote] arameter
>
> - depth       — 当前已放置的牛的数量 (也等于递归深度)
> - available   — 每种颜色当前仍可用的候选位置列表
> - placed_mask — 位掩码，bit c = 1 表示颜色 c 已放置

```cpp
    bool dfs(int depth, Available available, uint32_t placed_mask)
    {
        // ── 终止条件: n 头牛全部放置成功 ──
        if (depth == n_)
            return true;

        // ════════════════════════════════════════════════════════════════
        // 第一步: 动态 MRV — 选"候选位置最少"的未放置颜色
        // ════════════════════════════════════════════════════════════════
        //
        // 直觉: 候选少的颜色"瓶颈"大，应该优先处理。
        //       如果它有 0 个候选 → 立即回溯 (fail-fast)。
        //       如果它有 1 个候选 → 无需继续扫描，因为不可能更优。
        //
        // 对比静态排序 (构造时排一次):
        //   静态: 假设初始候选数多少 = 始终约束最紧 → 不成立，因为前向检查
        //         会动态删除候选，某个颜色可能中途变得极度受限。
        //   动态: 每层重新评估，始终选"当前最紧迫"的颜色 → 搜索树更窄。
        int best_color = -1;
        int best_count = INT_MAX;

        for (int c = 0; c < n_; ++c)
        {
            if (placed_mask & (1u << c))
                continue; // 已放置，跳过

            int cnt = (int)available[c].size();

            if (cnt == 0)
                return false; // 死胡同：无候选格
            if (cnt == 1)
            {
                best_color = c;
                break;
            } // 1 个候选已达最优
            if (cnt < best_count)
            {
                best_count = cnt;
                best_color = c;
            }
        }
       
        const auto &candidates = available[best_color];

        // ════════════════════════════════════════════════════════════════
        // 第二步: 遍历该颜色的每个候选位置
        // ════════════════════════════════════════════════════════════════
        for (const auto &[x, y] : candidates)
        {
            // ── 前向检查 (Forward Checking) ──
            //
            // 直觉: 常规回溯只检查"当前放不放得下"，前向检查额外检查
            //       "放了之后，剩余颜色还有没有格子可放"。
            //
            // 做法: 假设 (x,y) 放了牛，立即遍历所有未放置的颜色，
            //       从它们的候选列表中删掉与 (x,y) 冲突的位置。
            //       若任一颜色候选列表变空 → 此路不通，跳过。
            //
            // 效果: 在搜索树浅层就能发现死胡同，避免深层无效探索。
            //       与动态 MRV 形成正反馈循环:
            //         前向检查削减候选 → MRV 选到更紧的颜色 → 分支更少
            //
            // NOTE: 不能写成 Available next_available; (空声明)，必须拷贝。
            // 前向检查只修改"未放置且非当前选中的颜色"的候选列表，已放置的
            // 颜色需原样保留给下层递归。拷贝整份然后局部替换，比逐色判断省事。
            Available next_available = available;
            bool dead_end = false;

            for (int c = 0; c < n_; ++c)
            {
                if (c == best_color || (placed_mask & (1u << c)))
                    continue; // 跳过已放置和当前颜色

                const auto &old_list = available[c];
                Candidates new_list;
                for (const auto &[ox, oy] : old_list)
                {
                    if (!conflictsWith(ox, oy, x, y))
                    {
                        new_list.push_back({ox, oy}); // 不冲突，保留
                    }
                    // 冲突的 → 不加入 new_list，相当于删除
                }

                if (new_list.empty())
                {
                    dead_end = true; // 有颜色无格可放
                    break;           // 无需检查剩余颜色，直接跳出内层 for(c)
                }
                // NOTE: 这里不能在内层直接 continue 跳过，因为继续的目标
                // 会跳到下一个 c 而非下一个候选 (x,y)。因此用 dead_end 标志
                // + break 跳出内层，再由外层的 if (dead_end) continue 正确
                // 跳到外层 for 的下一个候选位置。
                //
                // move: 将 new_list 内部指针直接移交给 next_available[c]，
                // 避免逐元素拷贝 (O(1) vs O(n))。之后 new_list 变为空壳。
                next_available[c] = move(new_list);
            }

            if (dead_end)
                continue; // 剪枝，试下一个

            // ── 放置 ──
            board_[x][y] = 1;

            // ── 递归 ──
            uint32_t next_mask = placed_mask | (1u << best_color);
            if (dfs(depth + 1, move(next_available), next_mask))
                return true; // 找到解，向上传播

            // ── 回溯 ──
            // available 和 placed_mask 是值传递，自动恢复
            // board_ 需要手动回退
            board_[x][y] = 0;
        }

        return false; // 所有候选都失败
    }
```

## 复杂度分析

### 符号定义

| 符号      | 含义                                               |
| --------- | -------------------------------------------------- |
| $N$       | 颜色数 = 棋盘边长                                  |
| $D$       | 单色最大候选格数，$D \le N^2$，典型输入 $D = O(N)$ |
| $d$       | 递归深度 = 已放置牛数                              |
| $r = N-d$ | 剩余未放置颜色数                                   |
| $m_d$     | 深度 $d$ 时 MRV 选中颜色的候选数                   |
| $S_d$     | 深度 $d$ 时其余 $r-1$ 个未放置颜色的候选总数       |

### 单节点开销

```
MRV 扫描 ............ O(N)        遍历所有 N 色，跳过已放置，读取 size()  // 代码 L184-203
前向检查(每候选) .... O(S_d)      遍历其余 r-1 色，依次检验每个候选格     // 代码 L237-267
单节点合计 .......... O(N + m_d · S_d)
```

每节点中，MRV 扫描固定 $N$ 次迭代；对选中颜色的每个候选，前向检查遍历其余各色的全部候选列表，逐一调用 `conflictsWith`（$O(1)$）。

### 递推式

$$
T(r) = N + m \cdot \bigl(S + T(r-1)\bigr)
$$

展开为各层节点数 $\times$ 单节点开销：

$$
T(N) = \sum_{d=0}^{N-1} \underbrace{\Bigl(\prod_{j=0}^{d-1} m_j\Bigr)}_{\text{第 }d\text{ 层节点数}} \cdot \bigl[N + m_d \cdot S_d\bigr]
$$

### 最坏复杂度

无剪枝时各色始终保有 $D$ 个候选：$m_d = D$，$S_d = (N-d-1)D$，第 $d$ 层节点数 $D^d$。代入递推展开式：

$$
T_{\text{worst}}(N) = \sum_{d=0}^{N-1} D^d \cdot \bigl[N + D \cdot (N-d-1)D\bigr]
$$

拆分成 MRV 扫描（A）和前向检查（B）两部分：

$$
\begin{aligned}
A &= \sum_{d=0}^{N-1} D^d \cdot N
   = N \cdot \frac{D^N - 1}{D - 1}
   \approx N \cdot D^{N-1}
   &&\text{（等比数列求和）} \\[8pt]
B &= D^2 \sum_{d=0}^{N-1} D^d \cdot (N-d-1)
   &&\text{（算术-几何加权和）}
\end{aligned}
$$

对 $B$ 的内和式 $\sum_{d=0}^{N-1} D^d (N-d-1)$，$D^d$ 指数增长而 $(N-d-1)$ 线性衰减，尾部（$d \approx N$）主导。令 $k = N-1-d$，尾部近似：

$$
\sum_{k=0}^{\infty} D^{N-1-k} \cdot k
= D^{N-1} \sum_{k=0}^{\infty} \frac{k}{D^k}
= D^{N-1} \cdot \frac{D}{(D-1)^2}
= \frac{D^N}{(D-1)^2}
$$

故 $B \approx D^2 \cdot \frac{D^N}{(D-1)^2} = \frac{D^{N+2}}{(D-1)^2}$。

合并 $A$ 与 $B$：

$$
\begin{aligned}
T_{\text{worst}}(N)
&= N \cdot D^{N-1} \;+\; \frac{D^{N+2}}{(D-1)^2} \\[4pt]
&\sim \frac{D^{N+2}}{(D-1)^2}
\qquad\Bigl(\text{比值 }\frac{B}{A} \approx \frac{D^3}{N} \to \infty\Bigr) \\[4pt]
&= \Theta(D^N)
\end{aligned}
$$

> $D^{N+2}/(D-1)^2 = \Theta(D^N)$，因为 $\Theta$ 吸收常数因子 $D^2/(D-1)^2$。

典型 $D = O(N)$，故 $T_{\text{worst}} = \Theta(N^N)$。此问题泛化 N 皇后，属 **NP-complete**，最坏指数级不可避免。

### 期望复杂度（均匀分布假设下的启发式分析）

#### 每色候选损失的期望推导

设棋盘 $N \times N$，某剩余颜色 $c$ 有 $C$ 个候选格。在 $(x,y)$ 放置一头牛后，颜色 $c$ 损失多少候选？

冲突范围是第 $x$ 行（$N$ 格）、第 $y$ 列（$N$ 格）、四对角（$\le 4$ 格）。若颜色 $c$ 的候选格在棋盘上**均匀随机分布**，则：

$$
\begin{aligned}
\mathbb{E}[\text{行 }x\text{ 命中数}] &= C \cdot \frac{1}{N} &&\text{（}N\text{ 行中等可能）} \\[4pt]
\mathbb{E}[\text{列 }y\text{ 命中数}] &= C \cdot \frac{1}{N} &&\text{（}N\text{ 列中等可能）} \\[4pt]
\mathbb{E}[\text{对角命中数}] &= C \cdot \frac{4}{N^2} \approx 0 &&\text{（}N^2\text{ 格中 4 格）}
\end{aligned}
$$

初始 $C \approx \frac{N^2}{N} = N$（$N^2$ 格均分给 $N$ 色），故：

$$
\mathbb{E}[\text{每色每层损失}] \approx \frac{N}{N} + \frac{N}{N} = 2
$$

> 这是**期望值**，不是确定值。均匀分布假设意味着区域没有聚簇倾向——实际连通区域可能偏离此假设（见下文 § 假设边界）。

#### 均值场递推

在期望意义下，经 $d$ 层放置后每色候选数约 $D - 2d$。MRV 始终选中当前最紧迫的颜色（候选最少者），在均匀假设下各色候选数相近，故：

$$
m_d \approx \max(1,\; D - 2d)
$$

代入节点数递推（$D$ 为初始单色候选数，典型 $D \approx N$）：

$$
\text{nodes}_d = \prod_{j=0}^{d-1} m_j \approx \prod_{j=0}^{d-1} \max(1,\; D - 2j)
               = D \cdot (D-2) \cdot (D-4) \cdots
               = D!!
$$

其中 $n!!$ 为双阶乘（同奇偶项连乘）。$D = N$ 时搜索树节点总数：

$$
\text{nodes} \approx N!! \approx \bigl(\tfrac{2N}{e}\bigr)^{N/2}
\qquad\text{（偶数 }N\text{ 的 Stirling 型近似）}
$$

#### 节点数 → 总时间

单节点开销分两部分：

$$
\begin{aligned}
\text{MRV 扫描} &: \; O(N) &&\text{遍历所有 }N\text{ 色} \\[4pt]
\text{前向检查} &: \; m_d \cdot S_d &&\text{对 }m_d\text{ 个候选，各遍历其余 }(N-d-1)\text{ 色的候选列表}
\end{aligned}
$$

在均值场假设下 $S_d \approx (N-d-1) \cdot m_d$，故前向检查 $= O(m_d^2 \cdot N)$。

总时间 $=$ 各层节点数 $\times$ 单节点开销之和。当 $d$ 接近 $N/2$ 时，$m_d$ 缩减至 $1$（不再分支），而节点数已累积到 $N!!$。此后连续约 $N/2$ 层，每层均有 $\approx N!!$ 个节点。每节点中：

$$
\text{MRV } O(N) \;+\; \text{FC } O(1^2 \cdot N) = O(N)
$$

累加这些层：

$$
T_{\text{expect}}(N)
= \underbrace{\sum_{d=0}^{N-1} \text{nodes}_d \cdot O(N)}_{\text{MRV 在各层}}
\;+\;
\underbrace{\sum_{d=0}^{N-1} \text{nodes}_d \cdot O(m_d^2 \cdot N)}_{\text{FC 在各层}}
= O(N \cdot N!! \cdot N) \;+\; O(N \cdot N!! \cdot N)
= O\bigl(N!! \cdot N^2\bigr)
$$

> 直观：$N!!$ 个节点各做 $O(N)$ 工作，再乘上约 $N$ 层深度 → $N!! \cdot N^2$。

代入 $N!! \approx (2N/e)^{N/2}$：

$$
T_{\text{expect}}(N) \approx O\Bigl(\bigl(\tfrac{2N}{e}\bigr)^{N/2} \cdot N^2\Bigr)
$$

> 指数因子 $(2N/e)^{N/2}$ 主导增长，多项式 $N^2$ 在渐近意义上次要，但严格 $O(\cdot)$ 中不能省略。

#### 期望 vs 最坏 对比

|                | 最坏                            | 期望（均值场）                                                       |
| -------------- | ------------------------------- | -------------------------------------------------------------------- |
| **搜索树节点** | $\Theta(N^N)$                   | $O\big(N!!\big) \approx O\big((2N/e)^{N/2}\big)$                     |
| **总时间**     | $\Theta(N^N \cdot N^2)$         | $O\big(N!! \cdot N^2\big) \approx O\big((2N/e)^{N/2} \cdot N^2\big)$ |
| 指数（节点）   | $e^{N \ln N}$                   | $e^{\frac{N}{2} \ln \frac{2N}{e}}$                                   |
| $N=8$ 节点     | $\sim 1.7 \times 10^7$          | $8!! = 384$                                                          |
| $N=8$ 总时间   | $\sim 1.7 \times 10^7 \cdot 64$ | $384 \cdot 64 \approx 2.5 \times 10^4$                               |
| $N=10$ 节点    | $\sim 1.0 \times 10^{10}$       | $10!! = 3840$                                                        |
| $N=16$ 节点    | $\sim 1.8 \times 10^{19}$       | $16!! \approx 1.0 \times 10^7$                                       |

期望比最坏小约 $N^{N/2}$ 量级——MRV 选最紧迫颜色 + 前向检查提前剪枝的叠加效应。

#### 假设边界：均匀分布何时失效

上述推导假设各色候选格**均匀散布**于棋盘。实际连通区域可能偏离：

| 区域形状                          | 实际每层损失（每色）                | 与期望 $2$ 的偏离 |
| --------------------------------- | ----------------------------------- | ----------------- |
| 水平条带（一行）                  | $\approx 1$（仅列冲突）             | **减半**          |
| 竖直条带（一列）                  | $\approx 1$（仅行冲突）             | **减半**          |
| $\sqrt{N} \times \sqrt{N}$ 紧凑块 | $0$ 或 $2\sqrt{N}$（行/列命中与否） | **方差极大**      |
| 蛇形填充                          | 不规则                              | 不可预测          |

条带输入下损失减半 → 候选衰减更慢 → 搜索树比 $N!!$ 更宽。紧凑块输入下削减"团块化" → MRV 可能遭遇极端不均 → 某些分支极浅（fail-fast），某些分支深搜。两种偏离方向相反，$N!!$ 是均匀假设下的**中间估计**，不是严格上界或下界。

> 正因为区域形状影响显著，$N!!$ 只能以 $O$ 给出（启发式上界），不能用 $\Theta$（紧确界）。

---

### 规整输入下的严格上界

对区域形状做更强的结构性假设，可以得到可证明的上界。

若各色区域**充分铺展**（如每色占一行或一列），每次放置至少削减其余各色 $\ge 1$ 个候选（被占列/行与每色区域恰交于一点），则：

$$
m_d \le D - d,\qquad
\text{nodes} \le \prod_{d=0}^{N-1} (D - d) = \frac{D!}{(D-N)!}
$$

$D = N$ 时即为 $O(N!)$。由 Stirling，$N! \approx \sqrt{2\pi N}\,(N/e)^N$，对比最坏 $N^N$，比值 $\approx e^N / \sqrt{2\pi N}$，差距随 $N$ 指数扩大。

这是**可证明的上界**（在铺展假设下），而上一节的 $N!!$ 是**期望估计**。

| 5   | 水平条带   | 5   | <1μs | $\sim 10^1$ |
| 15  | 水平条带   | 15  | 56μs | $\sim 10^2$ |

对比：$15^{15} \approx 4.4 \times 10^{17}$，$15! \approx 1.3 \times 10^{12}$，$15!! = 2,027,025 \approx 2.0 \times 10^6$。实际节点数（$\sim 10^2$）远小于三者——条带输入下 MRV 内各色候选数始终相等、无"最紧迫"者可选，但行列消除本身已将分支因子从 $N-d$ 压到 $1$（$N$ 头牛恰好各占一行一列时每层唯一选择）。

---

### 小结

|        | 最坏                 | 期望（均值场）         | 规整输入（可证上界）      |
| ------ | -------------------- | ---------------------- | ------------------------- |
| 节点数 | $\Theta(N^N)$        | $O(N!!)$               | $O(N!)$                   |
| $N=8$  | $1.7 \times 10^7$    | $384$                  | $40320$                   |
| $N=15$ | $4.4 \times 10^{17}$ | $\sim 2.0 \times 10^6$ | $\sim 1.3 \times 10^{12}$ |

> [!note] 问题本身 **NP-complete**，最坏 $\Theta(N^N)$ 不可避免。$N!!$ 是均匀分布假设下的启发式期望估计（$O$ 非 $\Theta$）；$N!$ 是铺展区域假设下的严格上界。算法的实用价值来自 MRV + 前向检查在**典型输入**上将搜索树压缩到远低于理论界的规模。
